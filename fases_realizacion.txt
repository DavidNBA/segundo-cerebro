## Fase 3: Implementación del Módulo de Recursos (En Curso)
- **Objetivo:** Permitir a los usuarios organizar enlaces, y referencias a archivos y materiales de estudio, asociándolos a materias.
- **Tareas Detalladas:**
    1.  **Diseño de Datos (localStorage):**
        - Definir la estructura de un objeto 'recurso' (ej: { id: string, titulo: string, descripcion: string, tipo: string (Enlace, Libro, Video, Documento, Otro), url: string (opcional), materiaAsociada: string (opcional), etiquetas: array[string] (opcional) }).
    2.  **HTML (index.html):**
        - Crear la estructura básica para la sección de "Recursos" (`<section id="recursos">`).
        - Incluir un botón "Añadir Nuevo Recurso".
        - Definir un formulario (inicialmente oculto) para la creación/edición de recursos con campos para título, descripción, tipo (select), URL (si aplica), materia asociada (select) y etiquetas (input de texto, separadas por coma).
        - Crear un contenedor donde se mostrará la lista de recursos.
    3.  **JavaScript (script.js):**
        - Implementar funciones CRUD para los recursos en `localStorage`.
            - `guardarRecurso(recurso)`
            - `obtenerRecursos()`
            - `eliminarRecurso(idRecurso)`
            - `actualizarRecurso(recursoActualizado)`
        - Función para mostrar/ocultar el formulario de recursos.
        - Función para poblar el select de materias asociadas en el formulario de recursos.
        - Función para cargar un recurso existente en el formulario para edición.
        - Lógica para renderizar la lista de recursos. Cada recurso debe mostrar su título, tipo, descripción, URL (si es un enlace, hacerlo clickeable), materia asociada y etiquetas.
        - Event listeners para los botones del formulario de recursos y para los botones de "Editar" y "Eliminar" en cada recurso listado.
        - Actualizar la función `showSection` para que al hacer clic en "Recursos", se muestre la sección y se carguen/rendericen los recursos.
    4.  **CSS (style.css):**
        - Añadir estilos básicos para el formulario de recursos y la lista de recursos.
    5.  **Pruebas:**
        - Probar la creación de nuevos recursos de diferentes tipos.
        - Probar la visualización de la lista de recursos.
        - Probar la edición y eliminación de recursos.
        - Verificar que los enlaces son clickeables.
        - Verificar la persistencia de datos.
- **Estado:** Pendiente
